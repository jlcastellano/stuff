Ejercicio Java: Sistema de Gestión de Recursos para Biblioteca Digital

Contexto

Sistema que simula una biblioteca digital donde varios usuarios intentan acceder simultáneamente a diferentes recursos del sistema, demostrando problemas de concurrencia y bloqueos (deadlocks).

Componentes del Sistema

1. Recursos Compartidos:

  • Base de datos de libros: Almacena la información de todos los libros
  • Sistema de autenticación: Verifica credenciales de usuarios
  • Servidor de archivos: Almacena los archivos PDF de los libros

2. Tipos de Usuarios (Hilos):

Usuario Investigador (Tipo 1)

  1. Consultar base de datos
  2. Autenticarse
  3. Acceder al servidor

Usuario Normal (Tipo 2)

  1. Autenticarse
  2. Acceder al servidor
  3. Consultar base de datos

Usuario Administrador (Tipo 3)

  1. Acceder al servidor
  2. Consultar base de datos
  3. Autenticarse

Reglas de Funcionamiento

  • Cada recurso solo puede ser usado por un usuario a la vez
  • Un usuario debe mantener el recurso bloqueado mientras lo usa
  • No se puede saltar el orden establecido para cada tipo de usuario
  • Se debe simular un tiempo de uso para cada recurso
  • Después de usar todos los recursos, deben ser liberados

Objetivos del Ejercicio

1. Demostrar el Deadlock

Crear una situación donde los usuarios queden bloqueados mutuamente. Ejemplo:

  • Usuario 1 tiene la base de datos y espera autenticación
  • Usuario 2 tiene autenticación y espera servidor
  • Usuario 3 tiene servidor y espera base de datos

2. Implementar Soluciones

Implementar al menos una de estas estrategias:

  • Ordenamiento de recursos
  • Timeout
  • Detección y recuperación
  • Prevención

Requisitos de Implementación

  • Usar synchronized o Lock para el bloqueo de recursos
  • Implementar mecanismos de espera (wait/notify o similares)
  • Mostrar mensajes de estado del sistema
  • Documentar la solución implementada

Entregables Esperados

  1. Clase para cada tipo de recurso
  2. Clase para cada tipo de usuario (hilos)
  3. Clase principal que demuestre el deadlock
  4. Clase principal con la solución implementada
  5. Documentación explicando:
    • Cómo se produce el deadlock
    • Cómo lo soluciona tu implementación
    • Ventajas y desventajas de la solución elegida
