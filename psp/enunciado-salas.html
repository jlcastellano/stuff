Sistema de Reserva de Salas

Desarrollar un sistema de reserva de salas para una biblioteca utilizando programación concurrente en Java. El sistema debe cumplir con los siguientes requisitos:

  1. La biblioteca tiene 3 salas de estudio (numeradas del 1 al 3)
  2. Cada sala tiene una capacidad máxima de 4 personas
  3. Los estudiantes (representados por hilos) intentan reservar una sala aleatoria
  4. Un estudiante solo puede estar en una sala a la vez
  5. El tiempo de uso de cada sala es aleatorio (entre 1 y 5 segundos)
  6. Debe evitarse el interbloqueo

Requisitos específicos:

  • Crear una clase Sala que mantenga el control de su ocupación
  • Crear una clase Estudiante (que implementa Runnable) que intente acceder a las salas
  • Crear un gestor de reservas que evite interbloqueos
  • Implementar un método main que cree 10 estudiantes que intenten usar las salas

El programa debe mostrar mensajes indicando:

  • Cuando un estudiante intenta entrar en una sala
  • Cuando un estudiante consigue entrar en una sala
  • Cuando un estudiante sale de una sala
  • El número de personas en cada sala en cada momento

Consideraciones:

  • Usar synchronized para el control de acceso a las salas
  • Implementar wait() y notify() para la gestión de espera cuando una sala está llena
  • Establecer un orden en la reserva de recursos para evitar interbloqueos
