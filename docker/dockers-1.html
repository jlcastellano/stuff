1. Conceptos Fundamentales de Docker

1.1 ¿Qué es la Containerización?

La containerización es una tecnología que permite empaquetar una aplicación y todas sus dependencias en un contenedor virtual. Este contenedor puede ejecutarse de manera consistente en cualquier infraestructura que soporte Docker.

Imagina que tienes una caja que contiene todo lo necesario para que tu aplicación funcione: código, runtime, herramientas del sistema, librerías - básicamente, todo lo que instalarías en un servidor para ejecutar tu aplicación.

1.2 Diferencia entre Máquinas Virtuales y Contenedores

Máquinas Virtuales

  • Incluyen sistema operativo completo
  • Mayor consumo de recursos
  • Arranque más lento (minutos)
  • Aislamiento completo
  • Tamaño: GigaBytes

Contenedores

  • Comparten el SO del host
  • Consumo mínimo de recursos
  • Arranque rápido (segundos)
  • Aislamiento a nivel de proceso
  • Tamaño: MegaBytes

1.3 Arquitectura Básica de Docker

Docker utiliza una arquitectura cliente-servidor. Los componentes principales son:

  • Docker Daemon (dockerd): El servidor que gestiona los objetos Docker
  • REST API: Interface para comunicar con el daemon
  • Cliente Docker: Interface de línea de comandos (CLI)

1.4 Ventajas en el Desarrollo y Despliegue

  • Consistencia: "Funciona en mi máquina" deja de ser un problema
  • Velocidad: Desarrollo, testing y producción con entornos idénticos
  • Aislamiento: Las aplicaciones y sus dependencias no interfieren entre sí
  • Escalabilidad: Fácil replicación y distribución de contenedores
  • Versionado: Control preciso de versiones de la infraestructura
  • Portabilidad: Ejecuta en cualquier lugar que tenga Docker instalado

1.5 Conceptos Clave Adicionales

  • Imagen Docker: Plantilla de solo lectura con instrucciones para crear un contenedor
  • Contenedor: Instancia ejecutable de una imagen
  • Registry: Repositorio para almacenar y distribuir imágenes Docker
  • DockerFile: Archivo de texto con instrucciones para construir una imagen
