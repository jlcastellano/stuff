3. Comandos Básicos de Docker

3.1 docker pull

Descarga de imágenes

Descarga imágenes desde un registro (por defecto, Docker Hub)

docker pull [opciones] nombre_imagen[:tag]

Ejemplos:

docker pull ubuntu:latest
docker pull nginx:1.19
docker pull mysql:8.0

3.2 docker run

Ejecución de contenedores

Crea y arranca un nuevo contenedor basado en una imagen

docker run [opciones] nombre_imagen [comando]

Ejemplos comunes:

docker run -d nginx                    # Modo detached
docker run -it ubuntu bash             # Modo interactivo
docker run -p 8080:80 nginx           # Mapeo de puertos
docker run --name mi_web nginx        # Nombre personalizado

Las opciones más utilizadas son:

  • -d (detached mode)
  • -it (interactive terminal)
  • -p (port mapping)
  • --name (nombre del contenedor)

3.3 docker ps

Listar contenedores

Muestra los contenedores en ejecución

docker ps [opciones]

Variaciones:

docker ps           # Contenedores en ejecución
docker ps -a        # Todos los contenedores
docker ps -q        # Solo IDs
docker ps --format "table {{.ID}}\t{{.Names}}"

3.4 docker images

Listar imágenes

Muestra las imágenes descargadas localmente

docker images [opciones]

Ejemplos:

docker images
docker images -q    # Solo IDs
docker images --format "{{.Repository}}:{{.Tag}}"

3.5 docker stop/start

Control de contenedores

docker stop [contenedor]
docker start [contenedor]

Ejemplos:

docker stop mi_contenedor
docker start mi_contenedor
docker stop $(docker ps -q)    # Detener todos

3.6 docker rm/rmi

Eliminación de contenedores e imágenes

docker rm [contenedor]    # Eliminar contenedor
docker rmi [imagen]       # Eliminar imagen

Ejemplos:

docker rm mi_contenedor
docker rmi nginx:latest
docker rm $(docker ps -aq)    # Eliminar todos los contenedores

Para eliminar una imagen, todos los contenedores que la utilizan deben estar eliminados primero.

3.7 Comandos Adicionales Útiles

  • docker logs [contenedor]: Ver logs del contenedor
  • docker exec -it [contenedor] [comando]: Ejecutar comando en contenedor en ejecución
  • docker inspect [contenedor/imagen]: Ver información detallada
  • docker cp [origen] [destino]: Copiar archivos entre host y contenedor
