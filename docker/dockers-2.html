2. Componentes Principales de Docker

2.1 Docker Engine

El núcleo de Docker

Docker Engine es el componente central que actúa como motor de ejecución de contenedores. Consiste en:

  • Daemon: Proceso en segundo plano que gestiona objetos Docker
  • API REST: Interface para comunicación con el daemon
  • CLI: Interfaz de línea de comandos para interactuar con Docker
# Verificar la instalación y versión
docker --version
docker info

2.2 Docker Hub

Registro público de imágenes

Docker Hub es el registro más grande de imágenes Docker, ofreciendo:

  • Repositorios públicos y privados
  • Imágenes oficiales verificadas
  • Integración con GitHub/BitBucket
  • Automatización de builds

Las imágenes oficiales son mantenidas por Docker y los desarrolladores originales del software.

2.3 Docker Desktop

Entorno de desarrollo integrado

Aplicación para Windows y macOS que incluye:

  • Docker Engine
  • Docker CLI client
  • Docker Compose
  • Docker Content Trust
  • Kubernetes
  • Interfaz gráfica de usuario

2.4 Docker Compose

Orquestación de contenedores

Herramienta para definir y ejecutar aplicaciones multi-contenedor:

  • Define servicios en archivo YAML
  • Gestiona múltiples contenedores
  • Configura redes y volúmenes
version: '3'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      - MYSQL_ROOT_PASSWORD=example

2.5 Dockerfile

Archivo de configuración de imagen

Script que contiene instrucciones para construir una imagen Docker:

  • Define el entorno del contenedor
  • Especifica dependencias y configuraciones
  • Automatiza el proceso de construcción
FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

Los Dockerfiles siguen un conjunto de mejores prácticas para optimizar el tamaño y la seguridad de las imágenes.

2.6 Interrelación entre Componentes

Los componentes de Docker trabajan en conjunto para proporcionar un ecosistema completo:

  • Docker Engine: ejecuta los contenedores
  • Docker Hub: proporciona las imágenes base
  • Dockerfile: construye imágenes personalizadas
  • Docker Compose: orquesta múltiples contenedores
  • Docker Desktop: integra todo en una interfaz amigable
